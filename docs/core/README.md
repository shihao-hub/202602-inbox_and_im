## 💡 学习提示

### 重点关注

1. **内容与记录分离** - 这是站内信系统最重要的设计思想
2. **防止重复发送** - 唯一约束 + 代码检查双重保障
3. **软删除机制** - 数据可恢复，保留审计日志
4. **性能优化** - 批量插入、索引优化、分页查询

### 实践建议

阅读完本文档后，建议：
1. 运行项目，实际操作一下各个 API
2. 查看数据库表结构和数据
3. 尝试修改代码，添加新功能
4. 编写测试用例，验证理解
